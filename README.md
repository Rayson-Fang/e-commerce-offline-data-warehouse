电商离线数据仓库（E-Commerce Offline Data Warehouse）
基于 Hadoop 生态构建的电商订单离线数仓，实现数据采集、清洗、分层加工与指标计算，是企业级离线数仓的标准流程实现。
技术栈
存储：HDFS / MySQL
计算：Hive / Spark SQL
同步：Sqoop
数仓分层：ODS → DWD → DWS
技术：ETL、分区表、增量处理、小文件优化
项目背景
电商订单数据存在脏数据多、格式不统一、查询低效等问题，无法直接支撑业务分析。本项目搭建离线数仓，统一数据口径，提升查询性能，输出稳定可复用的统计指标。
项目功能
MySQL 业务数据通过 Sqoop 同步至 Hive ODS 层
数据清洗：空值处理、异常金额过滤、时间标准化
数仓分层设计（ODS/DWD/DWS），按日期 dt 分区
增量数据处理，避免重复
小文件合并优化，提升 HDFS 性能
用户维度聚合：订单量、消费总额等指标计算
指标验证，保证数据准确可用
项目成果
日均处理订单数据 10 万 +
分区与小文件优化使查询效率提升约 30%
完成全流程 ETL 开发，可直接用于业务分析
